Docker Installation and Basic Use

Prerequisites

Ensure you have a clean slate of Ubuntu server for this. The latest 2 LTS releases are supported. It's also a good idea to keep in mind that Docker Engine relies on containerd and runc, However these dependencies are bundled as containerd.io.

Uninstall old/conflicting versions/packages

The Docker Engine can conflict with some distributions if they maintain and include their own versions of the docker engine. For a debian or ubuntu based system uninstalling looks like the following:

sudo apt remove docker.io sudo apt remove docker-compose sudo apt remove docker-compose-v2 sudo apt remove docker-doc sudo apt remove podman-docker

Installing Docker

Below are the instructions to install Docker via its various methods.

First you will want to add Dockers official GPG Key

sudo apt-get update sudo apt-get install ca-certificates curl gnupg sudo install -m 0755 -d /etc/apt/keyrings curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg sudo chmod a+r /etc/apt/keyrings/docker.gpg

Then you will need to add Dockers official apt repository

echo \ "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \ $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \ sudo tee /etc/apt/sources.list.d/docker.list > /dev/null sudo apt-get update

Then you will need to install Docker

sudo apt-get install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

If you want/need a specific version you can do the following:

apt-cache madison docker-ce | awk '{ print $3 }'

It will output something like the following:

5:24.0.0-1~ubuntu.22.04~jammy 5:23.0.6-1~ubuntu.22.04~jammy ...

The following script requires root or sudo privileges to run and will attempt to detect your Linux distribution and version to configure your package management system for you.

The script isn't designed to upgrade an existing Docker installation. When using the script to update an existing installation, dependencies may not be updated to the expected version, resulting in outdated versions.

To install Docker Engine using the convenience script use the following command:

curl -fsSL https://get.docker.com -o get-docker.sh

If you wish to see the script run without making changes to your system you can run it with the --dry-run command. This will look like the following:

sudo sh ./get-docker.sh --dry-run
